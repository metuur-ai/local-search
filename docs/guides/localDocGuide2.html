Change Detection

local-doc automatically detects file changes before every query. It uses two strategies depending on whether your repo is a git repository.

commit Git repos (default)

When a registered repo has git initialized, local-doc uses git to detect changes. This is faster and smarter than filesystem scanning — git already knows exactly what changed.

1.On the first full scan, the current HEAD commit hash is stored in the database
2.On every subsequent query, the stored commit is compared against the current HEAD
3.If commits differ, git provides the exact list of changed spec and media files
4.Uncommitted changes (staged, unstaged, untracked) are also checked
5.Only the changed files are re-indexed — no full rebuild needed
$ local-doc search refund
(product: git changes detected — incremental update...)

  product: 2 files updated (incremental)

Results for "refund":
  ...
update Non-git repos (fallback)

Falls back to filesystem timestamp comparison using find -newer. If any spec file is newer than the database, a full rebuild is triggered. Works reliably but can't do incremental updates.

terminalEveryday Usage

Just type local-doc search.

Followed by what you're looking for. Here are the most common patterns.

Find documents containing a specific word, even if it's buried deep inside the file text.

$ local-doc search refund

Results for "refund":

  1. [product] payments/refund (.md)
     Refund flow
     tags: billing, refund, customer, payments

Use OR to find either word, NOT to exclude words, or filter by a specific project nickname.

# Search for A or B
$ local-doc search "refund OR authentication"
# Exclude words
$ local-doc search "billing NOT fraud"
# Only look inside the 'product' folder
$ local-doc search "billing" product

Not sure what you're looking for? List out everything you have.

# See all documents
$ local-doc list
# See project groups
$ local-doc projects
# See all tags
$ local-doc tags
# See stats
$ local-doc stats

Read the full content of a document right inside your terminal.

# Print the full document named 'refund'
$ local-doc read refund

Quick Start

Install

# copy to your PATH
cp local-doc-tool.sh /usr/local/bin/local-doc
chmod +x /usr/local/bin/local-doc

Requirements: bash + sqlite3 — both pre-installed on macOS and most Linux. Optional: git for smart incremental updates.

Quick start

# 1. Register your spec folders (auto-scans immediately)
local-doc repo add ./product-specs product
local-doc repo add ./platform-docs platform

# 2. Search — no manual scan needed, it just works
local-doc search refund

The index auto-rebuilds when you add/remove repos, and auto-detects when files change on your next search.

Frequently Asked Questions

Does local-doc require an internet connection? expand_more

No. Once your index is synced, all search and retrieval operations run entirely offline against your local SQLite database.

Which file formats are supported? expand_more

Markdown, YAML, JSON, TOML for indexed data. PDFs and images can be indexed via companion .md metadata files.

Can I use multiple repositories? expand_more

Yes. Run local-doc sync for each repository — all sources are merged into a single unified index and are searchable together.

How does the Claude Code Skill work? expand_more

The skill registers a local-doc query tool with Claude. When you ask a question, Claude calls the tool to search your local index and incorporates the results directly into its response.
