Setup in 2 Minutes

You only need to do this once. Open your computer's terminal (Command Prompt/Terminal app) and paste these lines.

1

Install the tool

cp local-doc-tool.sh /usr/local/bin/local-doc
chmod +x /usr/local/bin/local-doc
Works on Mac and Linux automatically. No extra downloads needed!
2

Tell it where your documents are

Add the folders on your computer that contain your specs or documents. Give each one a nickname (like "product" or "platform").

# Example: local-doc repo add [folder-path] [nickname]
local-doc repo add ./product-specs product
local-doc repo add ./platform-docs platform

It immediately indexes .md, .mdx, and .txt files. Images (.jpg, .png, etc.) and PDFs are also indexed when a companion .md sidecar with the same name exists alongside them.

Everyday Usage

Just type local-doc search followed by what you're looking for.

Find documents containing a specific word, even if it's buried deep inside the file text.

$ local-doc search refund
Results for "refund":
  1. [product] payments/refund (.md)
     Refund flow
     tags: billing, refund, customer, payments

Use OR to find either word, NOT to exclude words, or filter by a specific project nickname.

# Search for A or B
$ local-doc search "refund OR authentication"
# Exclude words
$ local-doc search "billing NOT fraud"
# Only look inside the 'product' folder
$ local-doc search "billing" product

Not sure what you're looking for? You can easily list out everything you have.

# See all documents
$ local-doc list
# See project groups
$ local-doc projects
# See all tags
$ local-doc tags
# See stats (how many files)
$ local-doc stats

Read the full content of a document right inside your terminal.

# Print the full document named 'refund'
$ local-doc read refund
# Refund flow

Process for managing customer refunds...
Step 1: Verify eligibility...

Images and PDFs are indexed via a companion .md sidecar with the same base name. Only one result appears per asset — the path points directly to the image or PDF.

# File layout — sidecar must share the same name
diagrams/
  architecture.png ← the image
  architecture.md ← companion (title, tags, description)
# Search finds it — result path points to the .png
$ local-doc search architecture
  1. [platform] diagrams/architecture (.png)
     System Architecture Diagram
     /path/to/diagrams/architecture.png
If no companion .md exists, the image or PDF is skipped and a warning is shown. The sidecar is never shown as a separate result — it only provides metadata for its asset.
Claude Code Skill

Turn Claude into a domain expert

`local-doc` ships with a custom skill for Claude Code. Instead of guessing or using general knowledge, Claude will follow a strict 3-step workflow to answer questions based on your actual project specs.

1

Search

Claude extracts domain nouns from your question and searches your index (e.g., local-doc search "payment eligibility").

2

Read

Claude automatically reads the top 2-4 matching specs to gather exact context, avoiding hallucination.

3

Reason

Claude synthesizes an answer that cites sources, connects dots across multiple specs, and explicitly flags gaps in your documentation.

Install the skill:

mkdir -p .claude/skills
cp -r /path/to/local-doc/skills/local-doc .claude/skills/

How it works behind the scenes

Your files are the boss

Your .md, .txt, images, and PDFs are always the source of truth. The database is a disposable cache — delete it any time.

Disposable Cache

It creates a tiny database file on your computer to make searches instant (~30 milliseconds). If things act weird, you can delete it safely—it rebuilds automatically.

Smart Git Updates

If your folder uses Git, the tool is incredibly fast. Instead of scanning every file every time, it asks Git exactly which files changed and only updates those.

Built for Agents

Type `local-doc json search [word]` to get perfect, clean data that automated bots and scripts can read easily.
